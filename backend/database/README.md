# SilentChurn AI – MongoDB Database Layer Documentation

This document describes the database design, collection schemas, index configurations, ingestion pipelines, and workflows for SilentChurn AI.

---

## 🏗️ MongoDB Atlas Architecture

SilentChurn AI uses **MongoDB Atlas** as its cloud database. The integration is built using `pymongo` and leverages a centralized, lazy-loaded connection manager in `backend/database/mongodb.py` to prevent redundant connection pooling.

Secrets (specifically the MongoDB URI connection string) are stored in the `.env` file and loaded dynamically using `python-dotenv`.

---

## 🗂️ Collections & Schemas

### 1. `raw_dataset`
Stores the original, unprocessed records of the Telco Customer Churn CSV dataset for auditability and historical tracking.
* **Fields**: Standard CSV string values (e.g. `customerID`, `gender`, `MonthlyCharges`, `TotalCharges`, `Churn`, etc.).

### 2. `cleaned_dataset`
Stores processed records after deduplication, data type correction, and cleaning.
* **Fields**: Cleaned, standard numeric and encoded indicators (e.g., `TotalCharges` coerced to float, nulls imputed, `customerID` kept for cross-referencing).

### 3. `model_metrics`
Stores the performance metrics of all trained model runs for comparison and performance tracking.
* **Schema**:
```json
{
  "_id": "ObjectId",
  "model_name": "XGBoost",
  "accuracy": 0.7970,
  "precision": 0.6507,
  "recall": 0.5080,
  "f1_score": 0.5706,
  "roc_auc": 0.8432,
  "created_at": "ISODate"
}
```
* **Indices**:
  * `created_at` (descending, for fast sorting of latest model metrics)
  * `model_name` (ascending)

### 4. `predictions`
Logs every inference result performed by the Customer Success Team or live API triggers.
* **Schema**:
```json
{
  "_id": "ObjectId",
  "user_id": "9999-TESTID",
  "prediction": "Churn",
  "churn_probability": 0.7313,
  "created_at": "ISODate"
}
```
* **Indices**:
  * `user_id` (ascending, for auditing user prediction history)
  * `created_at` (descending, for fetching latest customer predictions)

### 5. `recommendations`
Stores personalized retention plays generated by mapping customer-level SHAP explainability drivers to targeted CSM plays.
* **Schema**:
```json
{
  "_id": "ObjectId",
  "user_id": "9999-TESTID",
  "recommendation": "Target for 1-year or 2-year contract migration. Offer a 15% promotional discount on annual commitment.",
  "risk_driver": "Contract is 'Month-to-month'",
  "churn_probability": 0.7313,
  "created_at": "ISODate"
}
```
* **Indices**:
  * `user_id` (ascending)
  * `created_at` (descending)

## 🔄 Ingestion & Cleaning Workflow

```mermaid
flowcopy
[Original CSV] ──> Ingest to 'raw_dataset' collection
      │
      ├──> Clean data (deduplicate, numeric coercion, missing values)
      │
      ├──> Save local CSV ('datasets/cleaned_churn_data.csv')
      │
      └──> Ingest to 'cleaned_dataset' collection
```

1. **Ingest Raw**: `train_model.py` loads `datasets/WA_Fn-UseC_-Telco-Customer-Churn.csv`, cleans out the collection, and pushes all 7,043 documents into the `raw_dataset` collection.
2. **Ingest Cleaned**: The pipeline handles nulls in `TotalCharges` (setting to `0.0` for new users), converts columns to correct types, saves a local copy in `datasets/cleaned_churn_data.csv` for ML runs, and populates the `cleaned_dataset` collection.

---

## 🧠 Model Training & Prediction Workflow

1. **Model Evaluation logging**: During `train_model.py` execution, all classifiers (Random Forest, XGBoost) evaluate their scores. These scores are pushed directly to the `model_metrics` collection for audit comparisons.
2. **Prediction**: `predict.py` executes inference on real-time customer data and saves the probability, prediction classification, and timestamp under `predictions`.
3. **Personalized Action generation**: `recommendation_engine.py` calls the prediction model. If risk is high, it computes individual SHAP values via `shap_analysis.py`, isolates the primary risk factor, pairs it with an actionable marketing/customer success intervention, and inserts it into `recommendations`.
